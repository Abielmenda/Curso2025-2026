1. Dataset Analysis

1.1 Data Overview

The dataset contains air quality measurements in Madrid, including the following variables:

  • station_id: Identifier of the monitoring station (integer).
  • timestamp: Date and time of measurement (ISO 8601).
  • no2, o3, pm10, pm25: Pollutant concentrations (µg/m³).
  • latitude, longitude: Geographical coordinates of the station.

1.2 Data Quantities

Example subset analyzed: 5 rows, corresponding to 2 stations and 2 timestamps each. Values are within expected ranges: NO₂ (30–70 µg/m³), O₃ (20–60 µg/m³), PM10 (15–40 µg/m³), PM2.5 (10–30 µg/m³).

1.3 Schema

The dataset is structured as a simple CSV table with flat attributes, suitable for conversion into RDF triples using a defined ontology.

3. Resource Naming Strategy

3.1 URI Domain

URIs will use the base domain: http://linkeddata.airqualitymadrid.org/

3.2 URI Patterns

  • Ontology: http://linkeddata.airqualitymadrid.org/ontology/airquality#
  • Stations: http://linkeddata.airqualitymadrid.org/resource/station/{id}
  • Measurements: http://linkeddata.airqualitymadrid.org/resource/measurement/{station_id}_{timestamp}

3.3 Content Negotiation

URIs are designed to support content negotiation for HTML and RDF (e.g., via server configuration).
